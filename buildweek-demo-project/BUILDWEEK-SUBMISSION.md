# Build Week Submission Checklist

Use this page while completing the Devpost form.

## Required links and fields

- **Track:** Developer Tools
- **Project description:** Copy `SUBMISSION-NOTES.md`
- **Repository URL:** public repository, or a private repository shared with
  `testing@devpost.com` and `build-week-event@openai.com`
- **Demo video:** public YouTube link, under three minutes, with English audio
- **Primary Codex `/feedback` Session ID:** obtain it from the thread where most
  core project work was completed and paste it into Devpost

## Video must show

- the working TailTrail claims-validation demo
- what TailTrail does
- how Codex is used in the approved implementation step
- how GPT-5.6 is used through Codex
- the repeatable Evaluation Harness scenario and its claim boundary

## Repository must include

- `README.md` with setup, platforms, commands, and a no-rebuild judge path
- `DEMO-RUNBOOK.md` and `DEMO-PROMPTS.md` for the live recording
- `SUBMISSION-NOTES.md` for the submission description
- a valid open-source license and no secrets or private credentials

## Final review

- The demo video is public on YouTube and under three minutes.
- The repository can be opened and tested by judges.
- The README and video use the same commands and claims.
- The `/feedback` Session ID is from the representative build thread.
- No text claims exact token savings without measured telemetry.
- No text claims TailTrail replaces tests, CI, scanners, security review, or
  human review.
