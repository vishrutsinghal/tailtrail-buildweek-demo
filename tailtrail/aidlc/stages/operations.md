# Operations

Purpose: prepare production or support handoff when runtime behavior matters.

## Use When

- deployment is in scope
- migration or rollback is needed
- monitoring, alerting, or support ownership changes
- production data, privacy, or compliance risk exists

## Actions

- Record deployment steps and prerequisites.
- Record rollback or disablement path.
- Record migration and data validation notes.
- Record monitoring, alerting, logs, dashboards, and runbook updates.
- Record support owner and escalation path.

## Outputs

- `aidlc-docs/operations-notes.md`
- operations approval gate for standard or comprehensive depth

## Done When

- deploy and rollback paths are clear
- production validation is defined
- owners and follow-up are named
- operations approval is recorded when required
