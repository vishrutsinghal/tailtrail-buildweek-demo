# Prune Rules

Use pruning to keep long TailTrail sessions and project notes from growing stale.

## Keep

- current user goal
- explicit user requirements
- approved decisions
- active risks
- exact files being edited
- exact commands needed for validation
- current project map facts
- current AIDLC state when Phase 2 is active

## Prune Or Archive

- superseded plans
- duplicate summaries
- old tool output after a summary exists
- old failed approaches that no longer affect the work
- stale project facts whose invalidation conditions changed
- examples that are no longer relevant to the task

## Prune Check

Before carrying context forward, ask:

- Does this affect the next action?
- Is it still true?
- Is it exact material that must remain available?
- Is a path or cache key enough instead of full text?

If the answer is no, remove it from active context and keep only the source path or short summary.
